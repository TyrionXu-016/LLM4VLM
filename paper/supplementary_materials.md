## 1. 注意力可视化分析

### 1.1 跨模态注意力机制

我们的 VLN 模型使用跨模态注意力来融合语言指令和视觉特征。注意力权重展示了模型在不同字符位置上的关注程度。

**注意力计算**:
```
H_fused = CrossAttention(H_text, H_visual)
```

其中：
- `H_text`: 指令编码 [batch, seq_len, d_model]
- `H_visual`: 视觉编码 [batch, num_views, d_model]
- 注意力权重形状：[batch, seq_len, num_views]

### 1.2 典型样本注意力图

我们生成了 6 个典型样本的注意力可视化图，展示了不同类型指令的注意力模式：

| 样本 ID | 指令类型 | 指令内容 |
|---------|----------|----------|
| 0 | 直走 + 左转 | "直走 3 米，然后左转，看到过道就到了。" |
| 1 | 上下楼 + 左转 | "上楼/下楼后左转，走 4 米，看到椅子就到了。" |
| 2 | 走廊 + 右转 | "经过走廊，继续前进 11 米，在楼梯处右转。" |
| 3 | 起点 + 右转 | "从门口开始，走到窗户，然后右转走 7 步。" |
| 4 | 左转 + 直走 | "左转后直走，经过楼梯，走到尽头就是目的地。" |
| 5 | 直走 + 左转 | "直走 13 米，然后左转，看到楼梯就到了。" |

### 1.3 按指令类型的平均注意力

我们进一步分析了不同类型指令的平均注意力模式：
- **左转指令**: 模型在"左转"关键词上表现出更高的注意力权重
- **右转指令**: 类似模式，方向词获得显著关注
- **直走指令**: 注意力相对分散，距离词（如"3 米"）也获得较高关注

**可视化文件**:
- `attention_analysis_0.png` ~ `attention_analysis_5.png`: 单个样本注意力图
- `attention_by_type.png`: 按类型平均注意力图

## 2. 定性分析案例

### 2.1 总体统计

| 指标 | 数值 |
|------|------|
| 总样本数 | 200 |
| 成功案例 | 124 (62.0%) |
| 失败案例 | 76 (38.0%) |
| 边界案例 (3-4m) | 31 (40.8% 失败样本) |

### 2.2 按指令类型分析

| 指令类型 | 样本数 | 成功数 | 成功率 |
|----------|--------|--------|--------|
| 直走 | 138 | 91 | 65.9% |
| 含距离 | 128 | 80 | 62.5% |
| 左转 | 104 | 61 | 58.7% |
| 右转 | 96 | 63 | 65.6% |
| 走廊 | 63 | 43 | 68.3% |
| 上下楼 | 32 | 17 | 53.1% |
| 楼梯 | 22 | 17 | 77.3% |

**关键发现**:
- 楼梯相关指令成功率最高 (77.3%)，可能是因为楼梯作为显著地标容易被识别
- 左转指令成功率最低 (58.7%)，表明方向判断存在挑战
- 走廊指令表现良好 (68.3%)，说明线性路径较易导航

### 2.3 失败案例距离误差分布

| 误差范围 | 案例数 | 占比 |
|----------|--------|------|
| 精确 (<1m) | 0 | 0.0% |
| 良好 (1-2m) | 0 | 0.0% |
| 可接受 (2-3m) | 0 | 0.0% |
| 轻微偏差 (3-4m) | 31 | 40.8% |
| 中度偏差 (4-5m) | 21 | 27.6% |
| 严重偏差 (>5m) | 24 | 31.6% |

### 2.4 典型成功案例

**案例 1** (精确到达：0.24m):
- 指令："右转后直走，经过床，走到尽头就是目的地。"
- 置信度：0.9997
- DTW: 0.3969

**案例 2** (精确到达：0.28m):
- 指令："沿着走廊直走 8 米，左转进入卧室，椅子就在左边。"
- 置信度：0.9996
- DTW: 0.7396

**案例 3** (精确到达：0.29m):
- 指令："沿着走廊直走 10 米，左转进入客厅，楼梯就在对面。"
- 置信度：0.9995
- DTW: 0.1904

### 2.5 典型失败案例

**边界失败案例** (3-4m):
- 指令："经过过道，继续前进 11 米，在电梯处左转。"
- 距离误差：3.04m
- Oracle 成功率：失败

**严重失败案例** (>5m):
- 指令："上楼/下楼后右转，走 6 米，看到电梯就到了。"
- 距离误差：5.04m
- 置信度：0.9993

### 2.6 误差模式分析

**主要误差来源**:
1. **轨迹模拟简化**: 评估使用简化的直线轨迹模拟，而非真实路径规划
2. **视觉特征局限**: 使用单视角 ResNet 特征，缺乏多视角时序信息
3. **指令理解偏差**: 长距离指令 (>10m) 的理解精度下降
4. **方向判断困难**: 左转指令成功率低于右转和直走

## 3. 训练配置详情

### 3.1 模型架构

| 组件 | 配置 |
|------|------|
| 指令编码器 | Transformer Encoder (2 层) |
| 视觉编码器 | ResNet-50 + FC (2048→256) |
| 注意力头数 | 8 |
| 隐藏维度 | 256 |
| 词表大小 | 97 字符 |
| 最大序列长度 | 100 |

### 3.2 训练超参数

| 超参数 | 值 |
|--------|-----|
| Batch Size | 16 (有效 32，梯度累积 2 步) |
| 学习率 | 1e-4 (warmup + cosine) |
| Warmup 轮数 | 3 |
| 最大轮数 | 20 |
| 早停耐心值 | 5 |
| Dropout | 0.1 |
| 优化器 | AdamW |

### 3.3 训练进度

| Epoch | Train Loss | Val Loss |
|-------|------------|----------|
| 1 | 3.5815 | 3.5794 |
| 4 | 2.8873 | 2.6639 |
| 15 | 2.6303 | 2.6303 (最佳) |

**训练时间**: 约 1 分钟 (ResNet 特征)

### 3.4 评估指标

| 指标 | 定义 |
|------|------|
| SR (成功率) | 距离目标 < 3m 的比例 |
| SPL (效率加权) | 成功样本的路径效率加权 |
| Oracle SR | 轨迹中任意点到达目标的比例 |
| DTW (动态时间规整) | 预测轨迹与参考轨迹的相似度 |

## 4. 指令生成 Prompt 模板

### 4.1 基础 Prompt

```
给定一条室内环境中的路径，生成一条自然的中文导航指令。

路径：[起点] → 客厅 → 走廊 → 厨房 → [终点]
关键地标：沙发 (左侧)、餐桌 (中央)、楼梯 (右侧)
距离：约 15 米

生成的指令应包含：
1. 提及关键地标
2. 明确转弯方向
3. 包含距离估计
4. 听起来自然流畅
```

### 4.2 指令质量评估标准

| 维度 | 1 分 | 3 分 | 5 分 |
|------|------|------|------|
| 自然度 | 机器翻译痕迹明显 | 基本流畅，偶有生硬 | 母语级别自然 |
| 清晰度 | 方向模糊，难以理解 | 大部分清晰，少量歧义 | 方向明确，无歧义 |
| 可执行性 | 无法按指令导航 | 部分可执行 | 完全可执行 |
| 完整性 | 缺少关键信息 | 包含大部分信息 | 信息完整充分 |

### 4.3 LLM vs 机器翻译对比

| 方法 | 优秀率 | 平均分 |
|------|--------|--------|
| 机器翻译 | 20.0% | 4.28 |
| LLM 直接生成 | 76.7% | 4.61 |
